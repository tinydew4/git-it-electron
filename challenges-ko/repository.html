컴퓨터에 새 저장소 만들기.

저장소

저장소는 관련된 항목의 모음입니다. 우리의 경우, 소프트웨어를 만들때, 소프트웨어 프로젝트에 관련된 파일의 모음입니다. 프로젝트 폴더와 그 안의 모든 관련된 파일을 상상할 수 있습니다. 사실, 그것은 컴퓨터에서 언제든 볼 수 있는 것입니다. 때때로 줄여서 "repos" 라고 부릅니다.

당신의 프로젝트가 무엇인지 Git 에게 알려주면 Git 은 폴더의 모든 변경사항을 추적하기 시작할 것 입니다. Git 저장소로 만듭니다: 항목의 폴더는 Git 에 의해 추적됩니다. Git 은 파일이 추가되거나, 빠지거나, 한글자라도 변경이 되는 것을 추적합니다. 모든 것은 Git 에 의해 누가했는지와 언제 추적됐는지가 더해집니다. 소프트웨어에서, 변경사항 추적은 버전 제어로 불리는 것과 같습니다.

터미널 사용하기

터미널 (또는 Bash) 는 애플리케이션을 마우스로 누르는 대신 명령어를 입력하여 컴퓨터에게 명령을 내리는 것입니다. 파일 이름변경, 파일 열기, 새 폴더 생성, 디렉토리 (폴더) 간 이동과 입력 명령으로 실행되는 모든 것을 할 수 있습니다. 코드를 편집하기 위해 터미널을 떠나지 않고 (Vim 같은) 텍스트 편집기를 사용할 수 있습니다.!

게다가 컴퓨터를 탐색하면서, 터미널에서 명령줄 인터페이스 (CLI) 를 가지고 있는 프로그램도 사용할 수 있습니다. 터미널에서 명령으로 실행할 수 있다는 의미입니다. Git 은 그것 중 하나입니다. 명령의 첫 부분은 당신이 Git 명령을 사용하는 것을 컴퓨터에게 알려주는 것 입니다. 다음 부분은 Git 의 명령어와 그에 따른 다른 옵션입니다.

In Git-it 에서 컴퓨터를 탐색하기위한 몇가지 기본 명령줄 동작 (과 더불어 Git 명령) 을 배울 수 있습니다; 그것들은 단계에서 설명하고 있습니다.

저장소 생성

새 프로젝트 폴더를 만든 다음 Git 저장소로 초기화하는 것입니다. 이 도전에서 폴더 이름은 'hello-world' 로 할 것입니다. 하지만 이미 같은 이름의 폴더가 있다면 다른 이름을 선택할 수 있습니다 (그것은 일반적인 첫 프로젝트 이름입니다).

터미널을 열 때가 됐습니다!

터미널 창에서 이 명령들을 한번에 하나씩 입력하고 엔터/리턴을 입력하세요.

우선, 새 폴더를 만듭니다:
도움: mkdir 은 make directory 를 의미합니다

mkdir hello-world

그런 다음 해당 폴더로 이동합니다:
도움: cd 는 change directory change directory 를 의미합니다

cd hello-world

마지막으로, Git 에서 현재 폴더를 초기화 (추적 시작) 하라고 알려줍니다:

git init

마지막 명령은 "Initialized empty Git repository" 로 시작하는 무언가를 표시합니다. 다른 명령은 아무것도 표시하지 않습니다.

해내셨습니다! Git 저장소임을 다시 확인하려면, git status 를 입력하세요. 'fatal: Not a git repository...' 가 표시되지 않으면, 성공입니다!

저장소 디렉토리를 선택해주세요.
  • 새 폴더 만들기 (디렉토리 만들기)
    • mkdir <폴더이름>
  • 기존 폴더로 이동 (디렉토리 변경)
    • cd <폴더이름>
  • 폴더 내용 보기
    • ls
  • 폴더에 Git 켜기
    • git init
